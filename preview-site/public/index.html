Real use cases

图里就是日常会遇到的开发场景。

不是远程 SSH 的小屏硬撑，而是把 Codex 的关键动作重新整理成适合手机操作的控制面板。

Queue

灵感来了先排队，当前任务结束后自动接上。

开会、通勤、吃饭时想到补充需求，直接放进队列；需要调整顺序时置顶或编辑。

Runtime

状态不对时，一眼看到进程、队列和浏览器缓存。

右上角状态、运行时面板、停止入口、恢复检测放在一起，避免任务完成后还误以为仍在运行。

Skills

常用能力沉淀成 Skill，输入时可以快速带入。

Skill 管理异步扫描、本地缓存、变更提示，主聊天流程保持轻量稳定。

Feature map

围绕手机开发流做的功能，不是简单套壳终端。

01 会话持久化

服务端管理 Codex 子进程，页面关闭、浏览器退后台、网络断开后仍可回到原会话。

02 移动端消息体验

多行输入、图片上传、消息折叠、按轮查看、自动跟随底部，尽量减少手机屏幕浪费。

03 状态和安全控制

可查看运行时、终止当前命令、回收站删除、30 天登录策略、仅本机绑定部署。

04 自托管部署

适合 VPS、NAS、家用主机或远程开发机，推荐放在 HTTPS、VPN 或访问控制后面。

Install and update

OSS 安装，后续自动更新。

适合国内服务器：复制命令到服务器执行，安装完成后把服务放到自己的 HTTPS 域名后面即可。

01 复制安装命令

curl -fsSL https://welcome.ai.hehao.pro/install.sh | bash

02 自动完成部署

脚本会从 OSS 获取最新版本，安装依赖，创建 systemd 服务，并绑定到 127.0.0.1:7072。

03 默认自动更新

安装器会写入 OSS 更新源。后续有新版本时，控制台默认自动检查并安全升级。

04 需要时可关闭

如果你更想手动控制版本，可以在控制台设置里关闭自动更新。

# v1.4.0 - Public Launch Build

Codex Mobile Console is a self-hosted mobile control panel for persistent Codex development sessions. Codex runs on your server; your phone is a mobile-first control surface for switching sessions, sending prompts, checking runtime state, and stopping stuck runs.

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/twotwo7/codex-mobile-console/main/scripts/install.sh | bash
```

The installer:

- checks Linux, Node.js 20+, npm, git, curl, systemd, and Codex CLI
- clones or updates the app under `/opt/codex-mobile-console`
- installs npm dependencies
- creates a systemd service
- starts the app on `127.0.0.1:7072`
- prints the generated admin password

Put the app behind HTTPS, VPN, Tailscale, Cloudflare Access, or another trusted access layer before using it from a phone.

## Highlights

- Mobile-first PWA for controlling server-side Codex sessions
- Persistent Codex runs that survive terminal disconnects
- Recent, flat, directory-grouped, and trash session views
- Global Codex history discovery from `~/.codex/sessions`
- Queue support for prompts sent while Codex is already running
- Stop control and top-level run status indicator
- Runtime diagnostics for Codex process, browser state, service state, and frontend performance
- Image and file attachment support for prompts
- Skill manager backed by async local scanning and manual summary update prompts
- One-command installer and systemd service setup
- GitHub issue templates for bugs, feature requests, and deployment help

## Security Notes

This app can start Codex processes on your server. Treat it as a private control surface.

Recommended:

- keep `HOST=127.0.0.1` behind a reverse proxy
- expose only through HTTPS or a private access layer
- use a strong admin password
- do not publish a demo connected to a real server account
- do not commit `data/`, `runtime/`, `.env`, tokens, or private project content

## Known Limits

- Linux/systemd is the supported deployment path today.
- Codex CLI must already be installed and authenticated for the service user.
- This is designed for personal self-hosted use, not public multi-user hosting.
- Mobile browser backgrounding can still affect realtime connection behavior; the app uses reconnect and polling as recovery paths.
